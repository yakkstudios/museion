Museion Layered Architecture

The Museion dApp is not a monolith — it is a layered intelligence system with a clear responsibility boundary at each tier. Every layer communicates through typed, scoped interfaces. This is the Museion v1.0 target architecture that the current dApp is being built toward.

Frontend
Next.js 15 App Router + React
Server Components for static sections, Client Components for live HUD, SSE for real-time cabal alerts. Wallet Standard auto-discovery via @solana/react-hooks — no manual adapter arrays. Mobile-first, 375px baseline.
Web3 Auth
Wallet Standard · SIWS · JWT Bitfield
Sign-In With Solana ed25519 signature verification. Server-side API route (/api/gate-check) validates SIWS payload against Helius RPC balance. Issues scoped HttpOnly JWT with bitfield permissions (TIER_0–TIER_3). Frontend DevTools bypass architecturally impossible.
Backend
Vercel Edge Functions · Helius Webhooks
Edge API routes for gate-check and credit debit. Helius webhook subscriptions push wallet events to SSE streams. Exponential backoff + jitter on all RPC calls. Server-Sent Events bridge for HUD real-time feed.
AI Orchestration
LangGraph · RCR-Router · CrewAI
Python microservices. LangGraph state machines for council orchestration and HITL interrupts. RCR-Router slices context into role-specific sub-payloads for the 19 Kaize specialists. CrewAI for delegation velocity. Jarvis meta-persona synthesises final output.
Data & Indexing
PostgreSQL · pgvector HNSW · Alexandria
Local pgvector container with HNSW index on 768-dim Voyage AI embeddings. Full Alexandria schema with actor_id provenance, temporal domains, consensus levels, and deduplication. 24h ingestion cron. TypeScript + Python ingestion workers.
Smart Contracts
Anchor Rust · Solana · Intelligence Chain
$YST escrow, milestone-contingent vesting, OTC atomic swaps (SOL↔SPL), on-chain investigation attestations, 72-hour challenge windows. Sovereign SVM appchain (decoupled solana-svm crate) for forensic workloads — SOON Network model validated.
⚠ Critical Rule for Integrations

Never use the public Solana RPC (api.mainnet-beta.solana.com) for anything gate-critical. It's heavily rate-limited. Any balance read for a gate decision must go through the server-side /api/balance route backed by Helius.

Standard Component Pattern

Every section follows the same interface. If you're writing a component that integrates with the YAKK dApp, it must accept these exact three props — or none at all (for public sections like Services.tsx and Wren.tsx).

⚡ JSX Fragment Rule — Never Skip This

Any conditional render using {condition && ( that contains more than one sibling element must wrap children in a React Fragment <>...</>. Missing this causes Vercel build failures that are silent in local dev. Every Perplexity-generated section had this bug. Every one was fixed before shipping.

The Token Gate Architecture

The gate runs on three paths in sequence. The first path to succeed wins. This ensures holders are never locked out due to a single point of failure.

// Path 1 — Snapshot fast path (instant, no RPC) // lib/holders.ts — 17 known whale wallets // Catches known holders before any network call // Path 2 — Server-side Helius balance read // /api/balance?wallet={address} // Returns { balance: number }. In-memory 30s cache per wallet. // Path 3 — Retry of Path 2 after 800ms // Handles brief RPC transience // Gate-check route issues httpOnly cookie // POST /api/gate-check // → Verifies on Helius // → Returns 200 + sets yst_access cookie (STANDARD or WHALE) // → Returns 403 if balance below 250K // → Never returns 500 — always responds // middleware.ts intercepts server routes // Reads yst_access cookie before any gated API route
🐋 Whale Tier

Holders with 10,000,000+ $YST receive the WHALE cookie tier. This unlocks Whale Club — the most restricted section on the platform. Standard gate threshold is 250,000 $YST.

Environment Variables

All secrets live in Vercel's environment dashboard. Nothing is hardcoded — ever. If you're running a local dev build, create a .env.local file and add the following. Never commit it.

NEXT_PUBLIC_RPC_URL Helius RPC endpoint. Format: https://mainnet.helius-rpc.com/?api-key=YOUR_KEY. Used by SolanaProvider.tsx and all server routes.
HELIUS_API_KEY Private Helius key for server-side balance reads (/api/balance, /api/gate-check, /api/holders). Never expose client-side.
ANTHROPIC_API_KEY YAKKAI Coach — Claude API. Used in /api/ai.js. Wallet must pass gate check before any request is forwarded to this key.
DEV_WALLETS Comma-separated wallet addresses that bypass gate checks in development. Must also be listed in lib/constants.ts DEV_WALLETS array for client-side bypass.

Build Pipeline

Commits to main trigger an automatic Vercel deploy. There is no staging environment — main is production. The build must pass TypeScript, ESLint, and the Next.js build process before Vercel will promote a deployment to live.

Key build rules: no unused TypeScript variables (ESLint no-unused-vars is active), no duplicate const declarations after refactors, and all JSX fragments must be properly wrapped. Perplexity-generated output gets Hermes-validated before any commit.
