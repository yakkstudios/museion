Proactive Truth-Seeking
Scroll 015 · Anomaly Detection · YAKK Studios · April 2026
Most platforms wait to be asked. Museion watches. The detector runs even when no one is looking, and surfaces the pattern before the user has to find the words to describe it.

The Pull Model Is the Problem

Every existing chat-style tool is a pull model. A user suspects something is wrong, forms a question, types it, and waits. By the time the question has been formed, the liquidity has usually already moved. The detection is real, the timing is useless.

Proactive truth-seeking inverts the loop. Museion's anomaly engine is event-driven — it watches on-chain state in real time, fires when a pattern matches a signature, and notifies the user before the user has a question to ask. The question is answered before it is formed.

What Triggers a Signal

HOLDER CONCENTRATION
Gini above 0.85 or top five wallets controlling more than 60% of supply.
LP STATUS CHANGE
Liquidity unlocked, migrated, or partially removed outside the declared schedule.
KOL PRE-LOAD
Known KOL wallets accumulating before a promotion window.
CLUSTER CONVERGENCE
Multiple fresh wallets funded from a common source buying in the same minute.
TAX ROUTE SHIFT
Transfer tax redirected from staking pool to a new address.
BRIDGE ANOMALY
Large inbound transfer from a bridge with no matching announcement or liquidity event.

Each signal is deterministic. No probabilistic confidence levels on forensic claims. A rug pattern is either matched or it isn't. The response text is allowed to be cautious. The trigger logic is not.

"A system that waits to be asked is a system that has already decided its users are on their own."

From Signal to Scroll

Every confirmed signal is filed into Alexandria as a rug-detector scroll with an RD-XXX identifier. The scroll records the wallets involved, the on-chain trace, the matched precedent, and the outcome over the following seven days. The archive does not just serve future lookups — it verifies the detector.

A detector that cannot be scored against outcomes is a belief system. A detector that ships its own receipts is a truth-seeking instrument.

Why Determinism, Not Vibes

Probabilistic risk scores produce two kinds of wrong answer at once — false confidence when the score is high and false calm when the score is low. The determinism rule is not an engineering preference. It is a refusal to be part of a system that uses fuzzy language to launder fuzzy thinking.

Thresholds are public
Every trigger threshold is documented. A user can reproduce the decision from raw chain data.
Evidence is attached
Every alert cites the wallets, transactions, and time windows that caused it. Nothing is asserted without a link.
Outcomes are tracked
Seven-day follow-up is filed against every alert. False positives count. The detector has to be honest about its own record.
Rate limits are humane
Alerts are debounced. A noisy token doesn't produce ten alerts an hour. It produces one alert with a timeline attached.

The Social Ledger

Signals travel further when they are shared. Museion publishes a public feed of confirmed rug-pattern detections — not speculative accusations, but matched signatures with evidence. The feed is the layer partners can plug into. StakePoint can warn its stakers when a paired token lights up. StreamLock can adjust its reputation signals when a creator's promoted token matches a pattern. The user does not have to be inside Museion to be protected by Museion.
