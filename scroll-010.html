What Each Alliance Does in the dApp

📊

StakePoint — The Yield Layer

The StakePoint.tsx section is a public section (no $YST gate) — anyone can see staking data. The section calls the /api/stakepoint proxy route which forwards to stakepoint.app/api/ to avoid CORS. It surfaces lock data, pool stats, and staking stats for the SPT mint address. The proxy accepts a type param: locks | pool | stats.

🔐

StreamLock — The Supply Integrity Layer

LOCK's role is structural. It provides the locker infrastructure that the YAKK Cabal Detection Engine references when scoring tokens on the Trusted List. A token with provably locked liquidity via StreamLock scores better on the LP Lock dimension of the Cabal score. LOCK itself appears in the Screener and Terminal alongside YST, SPT, and SOL — surfacing real-time price data via GeckoTerminal pool BSGLNWFGMrVm1K7skYpDNkywZLG1tYdTfn44ZAq3Pw2M.

The LOCK Chart — Pair Address Fetching

The LOCK trading chart in the Screener and Terminal uses a TradingView embed. The pair address is not hardcoded — it is fetched live from /api/screener on component mount. This was a critical fix made on 2026-04-05 after the chart was breaking because the mint address was being used in the dex field instead of the pair address.

🔑 Chart Embed Rule

The TradingView chart embed requires the pair address in the URL — not the mint address. The current live pair for LOCK is BSGLNWFGMrVm1K7skYpDNkywZLG1tYdTfn44ZAq3Pw2M. Pair addresses can change if liquidity migrates — always fetch live from /api/screener. Never hardcode. Never use the mint address as a substitute. Additionally: never add a sandbox attribute to the iframe — TradingView requires web workers and cross-origin storage access that the sandbox attribute blocks silently.

Security Considerations for Integration Work

Integration Security Checklist

Before shipping any StakePoint or StreamLock integration code, verify: (1) All API calls go through the proxy route — never call stakepoint.app directly from client-side code (CORS will block it). (2) Staking amounts and positions are display-only — no write operations that move tokens can be initiated without explicit wallet confirmation via Solana Wallet Adapter. (3) Lock data displayed in the Trusted List or Cabal score is treated as advisory — it reflects on-chain state at fetch time and should be clearly timestamped. (4) Never hardcode pair addresses — always fetch live from /api/screener.
